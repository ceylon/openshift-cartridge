Welcome to Ceylon on OpenShift

Starting with this skeleton project, you can write your own Ceylon web application to replace the sample TODO list application and deploy it to OpenShift.

To begin, you must clone this project's Git repository, using git clone. You can find the URL of the Git repository for this project in the OpenShift web console, by clicking on the application you just created, and looking under Source Code.

For example, if you application is named sample, and your domain is domain, the Git URL will be something like this:

    git clone ssh://56316b4e2d527174890001c5@sample-domain.rhcloud.com/~/git/test.git/

Cloning the repository will create a directly named sample. In it, you'll find Ceylon source code located under sample/sources and static web content under sample/web-content.

You can edit the source code and static files and then push your changes by going to the sample directory and running:

    git commit -a -m "some commit message"
    git push origin master

Wait for OpenShift to finish deploying your changes, and then reload this page.

This project is configured to start by executing the toplevel method run() in the module ceylon.demo.net. You can change module name by editing the file ceylon.properties located in sample/.openshift/config/.

When you push changes to Ceylon source, the project is recompiled by OpenShift automatically. The Ceylon compiler will search for dependencies in Ceylon Herd, and in the module repository in sample/.openshift/config/modules/. So you can place any precompiled project-specific dependencies in this repository.
